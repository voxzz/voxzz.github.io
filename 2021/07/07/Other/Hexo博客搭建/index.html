Hexo博客搭建

发表于 2021-07-07 更新于 2021-07-17

Hexo是一款快速、简洁且高效的博客框架。Hexo使用Markdown解析文章，可以利用主题模板快速生成静态网页。

创建博客

安装Hexo

安装Hexo之前需要先安装下面两个软件。

  • Node.js
  • git

安装完成之后，便可以使用 npm 安装 Hexo。

建立站点

选择博客站点的根目录，在根目录打开终端，进行初始化。

部署博客

Hexo可以部署在自己的服务器上，也可以托管到公共代码仓库。

这里只说明托管到公共代码仓库的方法。

部署到 GitHub

建立仓库

在自己的 GitHub 中新建一个公共仓库，名字必须为<用户名>.github.io。

生成密钥

利用 Git 的工具生成 SSH 密钥。

过程全部默认即可，最后会提示密钥生成的位置，一般会在用户目录下的 .ssh 目录中。

打开 id_rsa.pub 文件，其内容便是 SSH 的密钥中的公钥。

设置密钥

打开 GitHub SSH Key 设置页面，位置为头像->Settings->SSH and GPG keys。

新建一个 SSH keys，将生成的公钥复制进去即可。

连接测试

在终端执行命令测试密钥是否生效。

如果出现下面提示，就是设置成功。

设置账号

在终端设置 GitHub 账号信息。

配置博客

打开博客根目录的配置文件_config.yml，找到deploy字段，修改配置。

上传部署

部署前需要先安装插件hexo-deployer-git。

执行命令hexo deploy或hexo d，将博客部署到 GitHub 。

访问https://<用户名>.github.io即可看到部署的博客页面。

需要注意的是，上传之后，页面不会立即改变，因为缓存的存在，会有一定延迟。

同时部署到 Gitee

因为国内访问 GitHub 的速度比较慢，所以可以同时上传一份到 Gitee。

建立仓库

在自己的 Gitee 中新建一个公共仓库，名字随意，不过出于后面设置域名方便一些，建议使用用户名作为仓库名。

建立仓库后，也设置下自己的 SSH 公钥，头像->设置->SSH公钥。

配置博客

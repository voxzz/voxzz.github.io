fd

题目描述

解题过程

SSH 登录，查看源码。

buf的内容需要为”LETMEWIN”才能通过验证，执行/bin/cat flag。

可以通过read函数进行输入，所以fd必须为0。

Linux 下的read函数有三种系统预定义的文件描述符：

  • 0：标准输入流。
  • 1：标准输出流。
  • 2：标准错误流。

所以只需要使用程序参数为 0x1234（4660）即可。

uaf

题目描述

解题过程

查看源码。

根据题目可知，需要通过 UAF 漏洞来调用Human类中的虚函数give_shell。

根据堆分配原则，当选项 3 释放两个对象后，再次分配的内存大小小于等于最后一个释放的对象大小，将会得到该对象内存的地址指针。

从 IDA 中可以看出，在调用operator new时，为对象分配的内存大小为 0x18。

所以选项 3 释放两个对象后，通过选项 2 申请小于 0x18 的空间时，将会得到w对象的内存指针，此时可以对w对象的内存进行读写操作。

根据 C++ 内存布局的特点：对象在内存开辟空间后，按照虚表指针、继承自基类的成员、类自身的成员的顺序进行存储。

由于程序并未开启 PIE，所以Woman类的虚表地址为 0x0000000000401550。

C++ 调用虚函数是通过虚表基地址 + 偏移地址来寻找虚函数地址，而不是像 GOT 表一样定位函数地址。所以选项 1 中调用introduce函数时，是将虚表地址 + 0x8 后找到introduce函数的，当虚表地址 - 0x8 后，实际得到的虚函数地址为give_shell函数地址。

选项 2 需要执行两次，因为m对象也被释放，所以需要在调用前，重新分配内存。

memcpy

题目描述

解题过程

查看源码。

只需要让程序执行完即可得到 flag。

测试时发现，fast_memcpy总是无法通过。查阅资料得知，原因在于movntps指令。

Moves the packed single-precision floating-point values in the source operand (second operand) to the destination operand (first operand) using a non-temporal hint to prevent caching of the data during the write to memory. The source operand is an XMM register, YMM register or ZMM register, which is assumed to contain packed single-precision, floating-pointing. The destination operand is a 128-bit, 256-bit or 512-bit memory location. The memory operand must be aligned on a 16-byte (128-bit version), 32-byte (VEX.256 encoded version) or 64-byte (EVEX.512 encoded version) boundary otherwise a general-protection exception (#GP) will be generated.

movntps指令要求目的地址必须按16字节对齐，而malloc分配的地址有可能不满足条件，程序将会报错。

根据堆结构的特点，堆头部大小为8字节，所以当申请8字节空间时，实际分配大小为16字节。所以若要保证地址按16字节对齐，则申请空间大小必须满足((size + 8) % 16) == 0。

unlink

题目描述

解题过程

查看源码。

从代码可以看出，需要利用的是堆溢出漏洞。通过从堆A覆盖堆B的前后向指针，来得到一次任意地址写入的机会。

查看程序开启的保护，只有 NX。所以可以通过精确修改函数返回地址来获取程序控制权。

程序会泄露两个地址，一个是A在栈中的地址，一个是A保存的堆的地址。

通过A在栈中的地址，可以计算出main函数的返回地址。通过A保存的堆地址，可以计算出B的前后向指针地址。

查看main函数栈布局，可以得到A距离返回地址的偏移为 0x18。

通过调试，可以得到三个堆块的地址。

可以计算出从A结构的buf成员距离B结构体的fd和bk成员的偏移分别为 0x18 和 0x20。

程序中已经准备好了system调用程序，地址为 0x080484EB。

根据unlink函数的逻辑，无论将FD和BK哪一个直接设置成返回地址，都将导致返回地址指向的函数代码被修改，可能会利用失败。

仔细分析main函数汇编代码，发现在main函数最后返回时并不是正常返回的。

这里返回地址的栈指针是由var_4变量中的值控制，所以直接修改返回地址并不能利用成功，因为这段代码会将eip恢复。但是通过修改var_4中的值，可以很好解决上面的问题。

将目的返回地址放在A结构体的buf数组中，将B结构体的FD指针指向栈中var_4变量地址，BK指针指向A结构体的buf数组。

在计算buf到B结构体便宜时，并不是 0x18，而是 0x10，因为buf为8字节，B结构体所在堆的头部为8字节，之所以调试时为 0x14，是因为调试时的堆与运行时堆不太一样。

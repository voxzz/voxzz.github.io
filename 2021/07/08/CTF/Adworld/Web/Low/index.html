攻防世界 - Web新手练习区

发表于 2021-07-08 更新于 2021-07-17 分类于 CTF

攻防世界 Web 新手练习区练习。

view_source

访问目标站点，提示如下，且右键无法使用。

直接”F12”查看源码，得到flag。

robots

直接访问目标站点，页面没有任何内容。

访问robots.txt文件，发现新文件。

访问f1ag_1s_h3re.php，得到flag。

backup

访问目标站点，

猜测index.php的备份文件为index.php.bak，访问下载index.php.bak文件。

打开i文件得到flag。

cookie

访问目标，提示”Cookie”。

F12在请求头中，查看cookie字段，存在一个cookie.png文件。

访问cookie.png文件，提示查看响应。

在响应包中得到flag。

disabled_button

访问目标，页面中按钮无法点击。

通过开发者控制台浏览页面元素，找到按钮代码。

删除input标签中的disabled属性，按钮将变为可用状态。

点击按钮，得到flag。

weak_auth

访问目标。

随便输入用户名和密码进行登录，提示用admin用户登录，得到用户名。

题目说密码是随手设置的，那么就是弱密码，尝试发现密码为123456。

simple_php

访问目标，是一段 PHP 代码。

从代码可知，需要两个参数a和b，且必须满足条件才能得到 flag。

a参数需要满足值为 0 且为 true，才能得到第一部分 flag。

b参数需要满足不为数组，且大小超过 1234，才能得到第二部分 flag。

根据 PHP 语言的弱类型特性，当'a' == 0时，结果为 true，所以a参数可以设置为”a”。is_numeric遇到数字和数字字符串将返回 true，但是当字符串中存在字符时，将不认为是数字，所以b参数可以设置为1235a。

get_post

访问目标，提示使用GET方式提交请求。

提交之后，提示以POST方式提交请求。

得到flag。

xff_referer

访问目标，提示 IP 地址必须为”123.123.123.123”。

X-Forwarded-For用于记录发送方的真实 IP 地址，在请求头中添加X-Forwarded-For字段，设置值为123.123.123.123。

referer用于记录请求来源地址，在请求头中添加referer字段，设置值为https://www.google.com。

webshell

访问目标。

使用任意 webshell 工具进行连接，打开目录即可获取flag。

command_execution

访问目标。

题目提示没有 waf，那么就是可以解析任意命令了，输入127.0.0.1;whoami测试。

查找 flag 文件127.0.0.1;find / -name "flag*"。

获取flag，127.0.0.1;cat /home/flag.txt。

simple_js

访问目标，提示输入密码。

查看源代码，得到一段 JS 代码。

经过分析，发现dechiffre函数中始终返回假密码，真密码位于fromCharCode字符串中。

先十六进制解码得到一段 ASCII 码。

然后转换成字符为”786OsErtk12”。

加上前缀，得到 flag 为”Cyberpeace{786OsErtk12}”。

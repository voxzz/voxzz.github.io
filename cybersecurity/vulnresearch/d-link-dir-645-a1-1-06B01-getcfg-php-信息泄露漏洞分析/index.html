漏洞分析

根据漏洞路由，查看 /htdocs/web/getcfg.php 页面的主要代码。

页面调用 is_power_user() 函数对当前请求的权限进行验证，通过后根据 SERVICES 参数的值读取 /htdocs/webinc/getcfg/ 目录下的文件内容。

在 /htdocs/webinc/getcfg/ 目录下有许多配置文件，其中 DEVICE.ACCOUNT.xml.php 可以读取到设备的账号信息。

分析 is_power_user() 可知，当 AUTHORIZED_GROUP 全局变量的值是为空或者为 “0” 时，表示验证失败，否则表示验证成功。

经过全局字符串搜索，猜测全局变量由负责 php 文件请求的 /usr/sbin/phpcgi 文件进行处理。

查看 cgibin 文件的 main 函数可知，phpcgi 文件由 phpcgi_main 函数负责处理。

在 phpcgi_main 函数中，根据请求方式的不同，将选择不同的解析函数。

程序调用 cgibin_parse_request 函数对 POST 请求的内容进行解析，然后将解析后的参数交给 PHP 页面进行处理。

当 POST 请求解析完成，程序将根据 Session 的有效性，设置 AUTHORIZED_GROUP 和 SESSION_UID 参数的值。

分析 cgibin_parse_request() 函数可知，程序以 & 作为分隔符对 POST 请求中的数据进行解析，然后使用 \n 作为解析后数据的分隔符，并将参数名称格式化为 _POST_XXXX 形式。

因此，当 POST 请求数据中有 A=a%0aAUTHORIZED_GROUP=1 参数时，将会被解析成 _POST_A=a\nAUTHORIZED_GROUP=1 形式的数据。

由于 AUTHORIZED_GROUP=1 比 AUTHORIZED_GROUP=0 位置靠前，所以导致 PHP 读取所需参数时，得到的是伪造的校验数据，而非正确的校验数据。从而实现权限认证的绕过。
